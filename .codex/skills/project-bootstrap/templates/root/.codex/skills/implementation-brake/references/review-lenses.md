# Implementation Review Lenses

Use only the lenses that most affect ship safety.

## Correctness

- Does the implementation satisfy the intended behavior?

## Regression Risk

- What existing behavior might this change break?

## Edge Cases

- Are failure paths, empty inputs, invalid inputs, or concurrency paths missing?

## Test Gap

- Is there proof for the changed behavior?

## Rigidity

- Is the implementation more complex or more coupled than necessary?
