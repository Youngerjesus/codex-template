# Task Master Planning

Use this skill after the spec and contract are approved.

## Rules

- Keep work unified at the spec boundary by default.
- Do not split one approved contract into multiple partial tasks unless the spec itself is sliced first.
- State dependencies explicitly.
- Keep task boundaries aligned with the spec and contract closure boundary.
